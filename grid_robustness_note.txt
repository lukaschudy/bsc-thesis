Grid Size Robustness Check
==========================
Does collusion require a coarse price grid?


MOTIVATION
----------
The supervisor's question is whether collusion in the Calvano et al. (2020) baseline
is a genuine emergent phenomenon or partly an artifact of having only m=15 discrete
prices. With a very coarse grid, firms have few choices and many of them already sit
above the Nash price, so "collusion" might emerge simply because there is not much
room to compete.


WHAT CALVANO ALREADY DID
-------------------------
Calvano (2020, Section 6.7) tests m in {15, 50, 100} and finds the average profit
gain Δ decreases with m but remains substantial (~70% at m=100). They conclude the
grid is not the primary driver of collusion.

The problem: they hold β fixed across all values of m. This means larger m leads to
a much larger state space that receives proportionally less exploration per cell.
Specifically, the state space has m^(n*k) states and m actions, so the total number
of Q-matrix cells scales as m^(n*k+1). At m=100 with β held at the m=15 value,
each cell is visited far fewer times on average. The algorithms may simply fail to
learn well rather than genuinely failing to collude. The comparison is not
apples-to-apples.


THE CONFOUND, PRECISELY
-----------------------
The expected number of visits to a suboptimal cell (ν) is:

    ν = (m-1)^n / { m^(k*n*(n+1)) * [1 - exp(-β*(n+1))] }

(Calvano eq. 21, footnote, with n=2, k=1 for the baseline.)

When m increases and β stays fixed, ν falls sharply. Lower ν means worse learning,
which on its own reduces the profit gain regardless of whether a finer grid makes
collusion harder. Calvano acknowledge this but do not correct for it.


THE CLEAN TEST
--------------
For each value of m we want to test, we solve for the β that keeps ν constant at
the baseline level. This holds learning quality fixed so that any change in Δ is
genuinely attributable to the finer price grid and not to worse exploration.

Rearranging the ν formula for β:

    β = -ln(1 - (m-1)^n / (ν * m^(k*n*(n+1)))) / (n+1)

Baseline parameters: n=2, k=1, m=15, β=4e-6 (the representative experiment in
Calvano). First compute the implied ν at the baseline, then use that ν to compute
the required β for each new m.

Step 1 - compute baseline ν:
    ν_baseline = (15-1)^2 / { 15^(1*2*3) * [1 - exp(-4e-6 * 3)] }
               = 196 / { 15^6 * [1 - exp(-1.2e-5)] }
               = 196 / { 11390625 * 1.19999e-5 }
               ≈ 196 / 136.69
               ≈ 1.43

(This is roughly consistent with Calvano's Table I experiment where they note
ν ≈ 20 at their chosen representative point α=0.15, β=4e-6. The exact value
depends on which formula variant is used; the key is to use the same formula
consistently across all m values.)

Step 2 - for each m, solve for β that gives the same ν:
    β(m) = -ln(1 - (m-1)^2 / (ν_baseline * m^6)) / 3

This must satisfy the constraint that (m-1)^2 / (ν_baseline * m^6) < 1,
which it does for all reasonable m.


PROPOSED EXPERIMENT GRID
--------------------------
m       β (adjusted)    State space size    Q-cells per agent
15      4.00e-6         225                 3375       ← Calvano baseline
25      TBD             625                 15625
50      TBD             2500               125000
100     TBD             10000             1500000

For each m, run 1000 sessions with the same α and other parameters as the
Calvano representative experiment (α=0.15, δ=0.95, logit demand, n=2, k=1).

Also run with Calvano's original fixed β for comparison, to reproduce their
finding and then show the corrected version side by side.


WHAT TO LOOK FOR
----------------
Case A - Δ stays high (≥70%) even at large m with controlled ν:
    Collusion is genuine and robust. The coarse grid is not doing the work.
    This strengthens the baseline before adding frictions.

Case B - Δ falls substantially as m grows even with controlled ν:
    A finer action space genuinely impedes coordination, possibly because
    with more prices it is harder to implicitly agree on a focal point.
    This is an interesting finding in its own right and has implications
    for real markets where prices are near-continuous.

Case C - Δ stays high with controlled ν but falls with fixed β:
    Calvano's original result was partly confounded by insufficient learning.
    The corrected test shows collusion is actually more robust than reported.


RELATIONSHIP TO THE FRICTION EXPERIMENTS
-----------------------------------------
This robustness check runs on the clean baseline (no frictions). It should be
completed and documented before introducing latency, noise, or asynchrony, for
two reasons:

1. It validates that our C++ implementation matches Calvano's results at m=15.
2. It fixes the baseline m that we carry into the friction experiments. If
   collusion is robust across m, we use the standard m=15 for the frictions
   (keeping the state space tractable). If not, that is itself a finding worth
   discussing in context.
